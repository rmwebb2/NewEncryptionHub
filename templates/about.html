About CipherGuard

CipherGuard is a web application that allows you to encrypt and decrypt data securely using industry-standard algorithms such as AES-256, RSA-2048, and ChaCha20. It’s designed for educational purposes as well as general usage, ensuring your sensitive information remains private.

How AES Works

AES (Advanced Encryption Standard) is a symmetric block cipher. This means the same key is used to encrypt and decrypt data. Key points:

  • Symmetric Key: Both sender and receiver share the same secret key.
  • Block Cipher: AES operates on fixed-size blocks of data (128 bits).
  • Key Sizes: Common key sizes include 128, 192, and 256 bits (we use AES-256 in this app).
  • Security: AES is considered very secure and is widely used by governments and organizations.

How RSA Works

RSA (Rivest–Shamir–Adleman) is an asymmetric encryption algorithm. This means it uses a pair of keys: a public key to encrypt and a private key to decrypt. Key points:

  • Public/Private Key Pair: The public key is shared openly, while the private key is kept secret.
  • Key Length: Common RSA key sizes include 1024, 2048, or 4096 bits. We use 2048 bits here.
  • Use Cases: Often used for secure key exchange, digital signatures, and encrypting small amounts of data.
  • Security: The strength of RSA relies on the difficulty of factoring large numbers.

How ChaCha20 Works

ChaCha20 is a modern stream cipher designed as an improvement over the Salsa20 family. It encrypts data byte-by-byte using a pseudo-random keystream generated from a 256-bit key and an 8-byte nonce. Key points:

  • Stream Cipher: Unlike block ciphers, ChaCha20 encrypts data continuously, which can be more efficient for certain types of data.
  • High Performance: Optimized for high-speed encryption on modern processors, making it well-suited for mobile and web applications.
  • Security: ChaCha20 is considered secure and is used in protocols like TLS as a robust alternative to AES.
  • Nonce Usage: Requires a unique nonce for each encryption operation to ensure the keystream is never reused.

Why We Use Multiple Algorithms

Each encryption algorithm offers its own strengths: AES is highly efficient for large volumes of data, RSA enables secure key exchange and digital signatures, and ChaCha20 offers a fast, secure stream cipher option that works well on various devices. By supporting multiple algorithms, CipherGuard provides flexibility and robust security for a range of encryption needs.
